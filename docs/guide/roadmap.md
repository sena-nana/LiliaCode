# 路线图

LiliaCode 的路线图围绕一个目标展开：把代码工程中的 Agent 会话变成可观察、可恢复、可调度的本地任务系统。

## 已落地能力

- 项目级管理：本地项目与 GitHub clone 项目可展示任务状态、活跃度、进度、会话与任务统计，并支持归档/排序。
- 任务化会话：会话持久化为 Task，可支持草稿提升、项目归属、置顶、优先级与生命周期管理。
- Milestone：里程碑与任务里程碑关系已可持久化管理，并已接入项目路线图页面。
- 插件与工具链接入：MCP、Skill、Plugin、Hook 已具备管理与运行时注入能力。

## 近期重点

- 补齐任务树与自动编排闭环，让任务依赖、阻塞状态、失败策略形成统一调度器。
- 统一可观测与治理面：将自动化、插件与 Codex/Hooks 事件在同一轨道聚合并提升运维可追踪性。
- 提升自动化编排场景下的恢复能力：恢复/重试、并发控制、策略降级与错误归因。

## 中期能力

- 任务树：管理父子任务、依赖关系和阻塞状态。
- 自动编排：根据任务状态、依赖关系和用户策略调度多个 Agent。
- 插件系统：把会改变 Agent 行为的能力做成可选择开启的插件。
- Memory：保存用户级和项目级记忆，并在合适时机辅助 Agent 使用。

## 长期方向

- Roadmap / Milestone：展示跨周、跨版本的工程进展。
- 辅助 Agent：在会话中运行低成本 Agent，实时监督和辅助主 Agent。
- 运行时协议收敛：只保留内置 Lilia 协议路径。

## 开发状态提醒

LiliaCode 仍处于快速变更阶段。本地数据库结构可能随功能调整而迁移或清空，请不要将它作为重要数据的唯一保存位置。
