# 路线图

LiliaCode 的路线图围绕一个目标展开：把代码工程中的 Agent 会话变成可观察、可恢复、可调度的本地任务系统。当前已经进入 `v1.0` 收尾阶段，路线图重心从扩展新功能转向稳定化和阶段性能力闭环。

## 已落地能力

- 项目级管理：本地项目与 GitHub clone 项目可展示任务状态、活跃度、进度、会话与任务统计，并支持归档/排序。
- 任务化会话：会话持久化为 Task，可支持草稿提升、项目归属、置顶、优先级与生命周期管理。
- 任务树基础：已支持父子关系、依赖维护、树形拖拽和阻塞状态提示。
- Milestone：里程碑与任务里程碑关系已可持久化管理，并已接入项目路线图页面。
- 插件与工具链接入：MCP、Skill、Plugin、Hook 已具备管理与运行时注入能力。
- Memory 基线：已支持手动保存用户级和项目级记忆，并在会话启动时按 Layer 1 基线注入。
- Android remote beta：PC HTTP bridge、二维码配对、active PC、任务收件箱、任务详情、timeline、composer 和关键操作已有实验性链路。

## v1.0 稳定化收尾

`v1.0` 的目标是让当前首发 beta 版本可信可用，不扩展新的大功能面。重点是把已出现的入口、协议和文档状态收束到一致，并把关键路径跑通到可发布。

- 核心对话链路：稳定发送、继续、恢复、时间线展示、权限交互、计划确认、文件上下文、斜杠命令和 IAB 基础路径。
- 任务与侧栏基线：稳定当前任务列表、置顶、归档、父子任务、依赖展示、阻塞提示和工作树相关基础操作。
- Roadmap / Milestone 基线：确保项目路线图、里程碑、任务关联和首发进度说明可以作为当前项目管理入口使用。
- Memory Layer 1：稳定手动保存、用户级/项目级记忆展示、会话启动基线注入和文档边界。
- 插件与工具链：收口 MCP、Skill、Plugin、Hook 当前管理面和诊断展示，不在 `v1.0` 扩展完整治理策略。
- 发布链路：完成 Windows 安装包、release workflow、release notes、安装验证和已知限制说明。
- Android 远控：作为 `v1.0-beta` experimental companion 能力纳入文档和验证；发布前需跑 `yarn android:verify`，并明确不承诺稳定完整远控产品。

## v2.0 会话工作台与团队模式

`v2.0` 的目标是把 LiliaCode 变成日常会话级工作台。重点从“能继续一条 Agent 会话”扩展到“能组织多个会话、多个 Agent 和任务依赖”。

- 大操作区 UI：重构聊天主界面，让对话、时间线、输入区和待处理交互拥有更大的稳定操作区。
- 完整侧边栏：补齐任务、产物、Memory、文件树、简单 Git 操作和项目上下文入口。
- 侧边栏产物：展示 Agent 输出的文件、报告、截图、构建结果、链接和后续动作。
- Memory 查看与 Layer 2：提供更完整的 Memory 浏览、过滤和会话机会窗口引导。
- 文件树与上下文：支持浏览项目文件树，并把文件、目录、图片等上下文稳定发送给 Agent。
- 简单 Git 操作：先覆盖状态、diff、stage/unstage、commit、分支和工作树基础操作；push、merge、rebase 等高风险操作继续要求显式确认。
- 任务创建与生命周期：让用户可以创建任务、设置状态、优先级、归属和完成条件。
- 任务依赖与分配：补齐依赖、阻塞、任务分配和团队模式的可操作闭环。
- 多 Agent 写作与调度：支持多个 Agent 参与同一目标，提供写作、审查、验证和整合流程。
- 自动编排主链路：把自动化框架接入会话主路径，形成调度、失败处理、恢复和降级策略。
- Claude Plugins / Subagents / 辅助 Agent：补齐 Claude Plugins 管理、Subagents 展示调度和低成本辅助 Agent。
- 对话级补全：补齐消息正文搜索、会话分叉、从指定 turn 继续、重生成和更多会话管理操作。
- Android 远控稳定化：把 `v1.0-beta` 的实验链路收敛为稳定远控入口，补齐 event stream、能力协商、前后台恢复、异常恢复和端到端回归。

## v3.0 项目级知识资产

`v3.0` 的目标是把项目层级信息从“辅助页面”升级为长期可维护资产。重点是让架构、原则、路线图和记忆能够持续沉淀，并反过来影响 Agent 工作。

- 项目总览高级度量：解释任务状态、活跃度、成本、风险和项目进展，而不是只展示聚合数字。
- 架构图一等资产：支持项目架构图生成、编辑、版本化、变更记录、回滚和任务引用。
- 美术 / 设计原则一等资产：把设计原则、视觉规范、产品约束和项目风格落到可查看、可引用、可演进的项目资产。
- 项目 Memory 高级检索：引入外置模型检索、质量评估、召回盲区检查和采纳率指标。
- Roadmap / Milestone 高级解释：支持跨周、跨版本、跨任务的进展解释、风险汇总和后续建议。
- 自动化高级能力：补齐 cron、webhook、重试、并发实例、子流程、远程执行和工作流复用。
- 插件 / 工作流市场化能力：让可复用插件、技能和工作流从本地管理面升级为可分发资产。
- 远控与多设备协作：在 Android remote 稳定化之后评估多设备、PC-PC 和更复杂远程协作。

## 开发状态提醒

LiliaCode 仍处于快速变更阶段。本地数据库结构可能随功能调整而迁移或清空，请不要将它作为重要数据的唯一保存位置。
